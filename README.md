# Конвертер школьных Excel-журналов

Настольное приложение на Python для преобразования неструктурированных школьных Excel-журналов из широкого формата в нормализованный плоский вид. Проект ориентирован на реальные журналы, в которых встречаются объединенные ячейки, плавающая структура колонок, различия в названиях предметов и временные замены учителей.

## Описание проекта

Во многих школах журналы выгружаются в Excel в неудобном для анализа формате: даты распределены по колонкам, уроки привязаны к визуальной разметке, а итоговые оценки и данные по посещаемости находятся рядом с оперативными отметками. Такой файл сложно загружать в BI-системы, базы данных, Python-аналитику и ETL-пайплайны.

Этот проект решает задачу нормализации таких журналов:

- извлекает уроки, темы и домашние задания;
- распознает оценки и отметки посещаемости;
- вычисляет номер триместра по исходной бизнес-логике;
- корректно обрабатывает объединенные ячейки дат;
- поддерживает JSON-переопределения учителей, если в выгрузке указан замещающий педагог;
- сохраняет результат в единый Excel-файл с плоскими таблицами.

Итоговый файл формируется в двух листах:

- `Оценки и посещение` — длинный формат по ученикам и урокам;
- `Уроки` — справочник уроков с датой, темой и домашним заданием.

### Слои

`src/gui/`
Графический интерфейс на PyQt6. В этом слое нет бизнес-логики парсинга Excel. GUI работает только через сервисный слой, сигналы Qt и стандартный `logging`.

`src/core/`
Ядро приложения. Здесь находятся:

- конвертация листа журнала в DataFrame;
- пакетная обработка папки Excel-файлов;
- модели домена;
- сервисный сценарий запуска конвертации;
- репозиторий JSON-переопределений учителей.

`src/utils/`
Константы, вспомогательные функции, регулярные выражения и переиспользуемые настройки.

### Принципы проектирования

- сохранение исходной бизнес-логики без изменения алгоритмов;
- явное разделение UI и обработки данных;
- строгая типизация;
- читаемые русскоязычные docstrings;
- единый стандарт логирования;
- возможность вывода логов как в консоль, так и в `QTextEdit` через кастомный `logging.Handler`.

## Локальная установка

### 1. Клонирование репозитория

```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd School№88
```

### 2. Создание виртуального окружения

```bash
python -m venv .venv
```

Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Linux / macOS:

```bash
source .venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Запуск приложения

Основной способ:

```bash
python main.py
```

Для обратной совместимости также поддерживается запуск:

```bash
python converter.py
```

## Сборка в `.exe`

Для упаковки приложения в исполняемый файл под Windows удобно использовать `PyInstaller`.

### 1. Установка PyInstaller

```bash
pip install pyinstaller
```

### 2. Сборка в папку `dist/`

```bash
pyinstaller --noconfirm --windowed --name SchoolJournalConverter main.py
```

После выполнения команды готовый файл будет находиться по пути:

```text
dist/SchoolJournalConverter/SchoolJournalConverter.exe
```

### 3. Сборка в один файл

```bash
pyinstaller --noconfirm --windowed --onefile --name SchoolJournalConverter main.py
```

После выполнения команды готовый файл будет находиться по пути:

```text
dist/SchoolJournalConverter.exe
```

### 4. Что важно учитывать

- Сборку рекомендуется выполнять из активированного виртуального окружения.
- Перед сборкой желательно убедиться, что приложение запускается локально через `python main.py`.
- Если в дальнейшем потребуется иконка приложения, информация о версии или тонкая настройка ресурсов, имеет смысл перейти на сборку через `.spec` файл.

## Руководство по использованию

### Загрузка файлов

1. Откройте приложение.
2. На вкладке `Загрузка файлов` выберите папку с журналами `.xlsx` или перетащите файлы в окно.
3. Нажмите `Начать работу`.
4. После завершения приложение создаст итоговый Excel-файл рядом с исходной папкой.

### Актуализация учителей

Если в журнале указан замещающий учитель, можно заранее настроить правило замены:

1. Перейдите на вкладку `Актуализация учителей`.
2. Укажите класс.
3. Введите часть названия предмета.
4. Укажите ФИО основного учителя.
5. Нажмите `Добавить`.

Правила сохраняются в файл `teacher_overrides.json` в корне проекта и автоматически применяются при обработке.

### Отчет и логирование

Во вкладке `Отчет` отображается журнал выполнения:

- какие файлы обрабатываются;
- сколько записей извлечено;
- были ли пропуски или ошибки;
- куда сохранен итоговый результат.

## Особенности бизнес-логики

Ключевое требование проекта — сохранение существующего поведения преобразования данных. Поэтому в refactoring-версии специально сохранены:

- алгоритм определения номера триместра;
- логика привязки итоговой оценки к триместру;
- обработка сокращенных названий месяцев;
- восстановление дат из объединенных ячеек;
- правила извлечения оценок и посещаемости;
- механизм teacher mapping через JSON;
- генерация идентификаторов уроков.
